10. PERSISTENT SESSION — FILE-BASED (Replaced Cookies)
================================================================================

  PROBLEM:  CookieManager from extra_streamlit_components was buggy —
            caused duplicate key errors, caching warnings, and sessions
            were lost on every page refresh.

  SOLUTION: Replaced with file-based sessions + st.query_params.

  NEW FILE: auth/session_store.py
  --------------------------------------------------------------------------
    save_session(user, access, refresh) -> session_id
        - Writes JWT + user data to sessions/{uuid}.json
        - Returns session_id for URL storage

    load_session(session_id) -> dict | None
        - Reads session file, returns data or None

    delete_session(session_id)
        - Deletes session file on logout

  HOW IT WORKS:
    LOGIN   → save_session() → st.query_params["sid"] = session_id
    REFRESH → read st.query_params["sid"] → load_session() → restore state
    LOGOUT  → delete_session() → st.query_params.clear()

  MODIFIED: auth/middleware.py
    - Simplified: no cookie logic, returns tokens from login_user()
    - logout_user() only clears session_state

  MODIFIED: app/streamlit_app.py
    - Removed extra_streamlit_components dependency
    - Added top-level session recovery block
    - Login/logout now use file-based session store
